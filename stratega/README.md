# STRATEGA-LAM - Торговый агент

Сайт торгового агента с витриной товаров и услуг, системой сбора заявок и запросов цен.

## Технологии

- **Next.js 14** (App Router) - React фреймворк
- **TypeScript** - типизация
- **Prisma** - ORM для работы с БД
- **SQLite** - база данных (для продакшена можно перейти на PostgreSQL)
- **Tailwind CSS** - стилизация
- **React Hook Form + Zod** - формы и валидация
- **Radix UI** - UI компоненты

## Установка

1. Установите зависимости:
```bash
npm install
```

2. Настройте переменные окружения:
```bash
cp .env.example .env
# Отредактируйте .env файл
```

3. Инициализируйте базу данных:
```bash
npm run db:generate
npm run db:push
```

4. Запустите dev сервер:
```bash
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000)

## Структура проекта

- `/app` - страницы Next.js (App Router)
- `/components` - React компоненты
- `/lib` - утилиты и конфигурация
- `/prisma` - схема базы данных

## Основные функции

### MVP
- ✅ Каталог товаров с категориями
- ✅ Витрина услуг
- ✅ Система заявок/RFQ
- ✅ Админ-панель (базовая)
- ✅ SEO оптимизация (sitemap, robots.txt)

### Фаза 2 (планируется)
- CRM интеграция
- Мультиязычность
- Личный кабинет клиента

## Админ-панель

Доступ: `/admin`

**Внимание:** В MVP версии аутентификация не реализована. Добавьте её перед деплоем в продакшен.

## База данных

Используется Prisma ORM. Для изменения схемы:

1. Отредактируйте `prisma/schema.prisma`
2. Выполните миграцию:
```bash
npm run db:migrate
```

Или для разработки:
```bash
npm run db:push
```

## Деплой

1. Настройте переменные окружения на сервере
2. Соберите проект: `npm run build`
3. Запустите: `npm start`

Для продакшена рекомендуется:
- Использовать PostgreSQL вместо SQLite
- Настроить резервное копирование БД
- Добавить аутентификацию в админку
- Настроить SMTP для уведомлений
- Добавить reCAPTCHA для защиты форм
